<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser file input -> API payload | User-selected files and metadata are untrusted |
| Client role state -> admin UI | Local role data controls presentation but cannot authorize server writes |
| Local palette values -> preview/style props | Tenant-controlled colors enter inline style values |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-09 | Elevation of Privilege | hidden/redirected admin UI | mitigate | Client redirects members, but every API mutation remains server-enforced; UI never treats hiding as authorization |
| T-02-10 | Denial of Service | oversized browser upload | mitigate | Reject >5 MB before FileReader/request; server independently repeats validation |
| T-02-11 | Tampering / CSS injection | hex inputs | mitigate | Accept only strict six-digit hex before preview/save; never interpolate arbitrary CSS text |
| T-02-12 | Information Disclosure | error display | mitigate | Render server error strings as React text, never HTML; no raw image/base64 in alerts/logs |
</threat_model>
