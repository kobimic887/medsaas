<objective>
Prove — empirically, before any Phase 2 feature code depends on it — that `node-vibrant` and `sharp` install under Bun on linux/arm64 inside the `oven/bun:1.3.14-slim` production container and can extract a structured colour palette (dominant + accent hex) from a raster logo. Probe SVG -> palette as a recorded finding (not a gate). Record a GO/NO-GO verdict.

This is a throwaway, container-isolated de-risking spike (COMPAT-01 only) mirroring the v2 `spike/` spike-before-commit precedent. It wires nothing into `server/` and ships no production code.

Purpose: Resolve the one open question delegated to (absent) research — the current Bun-compatible `node-vibrant` major version and its correct import shape — empirically, and confirm `sharp`'s arm64 prebuilt binary + libvips availability under the slim base, so Phase 2 builds on a proven foundation.
Output: A numbered spike script, a minimal spike Dockerfile, a container-run harness, a small SVG fixture, and a GO/NO-GO findings doc.
</objective>
