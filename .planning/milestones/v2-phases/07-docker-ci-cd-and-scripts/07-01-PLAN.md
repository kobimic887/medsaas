Output: A Bun-based `Dockerfile` and a durable `ROLLBACK.md`.

    Rewrite the root `Dockerfile` keeping the SAME two named stages and the SAME frontend→api dist copy, swapping the runtime to Bun.

Copy the lockfile alongside the manifest: `COPY client/package.json client/bun.lock ./` (bun.lock exists per the dual-lockfile rule). Replace `RUN npm ci` with `RUN bun install --frozen-lockfile`. Keep `COPY client ./`. Replace `RUN npm run build` with `RUN bun run build` (this invokes the existing Vite build via Bun, consistent with Phase 6 PKG-03).

    Api stage: change `FROM node:22-alpine AS api` to `FROM oven/bun:1.3.14-slim AS api`. Keep `WORKDIR /app/server`. Copy `COPY server/package.json server/bun.lock ./`. Replace `RUN npm ci --omit=dev` with `RUN bun install --frozen-lockfile --production`. Keep `COPY server ./`. PRESERVE the line `COPY --from=frontend /app/client/dist ../client/dist` exactly (this is the key link that lets the Bun server serve the built frontend). Keep `ENV NODE_ENV=production`, `ENV FRONTEND_DIST=../client/dist`, and `EXPOSE 3000`. Change the final line `CMD ["node", "index.js"]` to `CMD ["bun", "index.js"]`.

    Pin the base image to `oven/bun:1.3.14-slim` (the exact tag proven on arm64 in the Phase 4 spike) — do NOT use `oven/bun:latest`. Use `bun install --frozen-lockfile` (not the spike's bare `bun install`) so the build is reproducible from the committed lockfile and cannot silently resolve drifted transitive versions inside the image.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| base image → built image | Untrusted upstream image layers enter the production artifact |
| committed lockfile → installed deps | Dependency resolution determines what code ships in the image |
| build args / source → image layers | Secrets must not be baked into image layers |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | base image tag (`FROM`) | mitigate | Pin `oven/bun:1.3.14-slim` exactly (no `latest`); tag asserted by grep in Task 1 acceptance criteria |
| T-07-02 | Tampering | dependency resolution in image build | mitigate | `bun install --frozen-lockfile` installs only what the committed `bun.lock` pins; floating transitive versions cannot enter the image |
| T-07-03 | Information Disclosure | secrets in image layers | accept | No `COPY .env` / no `ENV` secret in Dockerfile; runtime secrets are injected by `docker-compose.box.yml` via `env_file: .env` (existing, correct control — not modified here) |
| T-07-SC | Tampering | npm/bun installs | mitigate | No NEW packages added in this plan; install is from existing committed lockfiles only. RESEARCH Package Legitimacy Gate N/A (zero new deps) |
</threat_model>
