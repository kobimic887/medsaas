<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Developer docs to shell execution | README and CLAUDE commands drive what humans and agents run locally. Incorrect docs can execute the wrong package manager or fallback path. |
| Bun build command to generated frontend bundle | `bun run build` invokes Vite and creates files served by the Express static host. |
| Built frontend bundle to Bun unified server | `client/dist` crosses into `server/index.js` static serving via `FRONTEND_DIST=../client/dist`. |
| Verification report to future maintainers | The phase report becomes the source of truth for what was verified and which lockfiles/scripts are expected. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-05 | Spoofing | Documentation command paths | mitigate | Document exact Bun default commands and exact npm/Node fallback commands; verify docs contain both command families. |
| T-06-06 | Tampering | `client/dist` generated by Bun-invoked Vite | mitigate | Verify `bun run build` runs the existing Vite build and then smoke-test static serving under Bun with `--assert-static`. |
| T-06-07 | Repudiation | Phase 6 verification evidence | mitigate | Create `BUN-PACKAGE-MANAGEMENT.md` with command names, lockfile inventory, and static-serving assertions. |
| T-06-08 | Denial of Service | Fallback build path | mitigate | Run `npm run build:node` so the documented npm fallback remains executable after the Bun script migration. |
| T-06-SC | Tampering | Package-manager/lockfile supply chain | mitigate | Docs require both Bun and npm lockfile families, `lockfiles:refresh`, and exact fallback commands; report records verified lockfile inventory and command results. |
</threat_model>
