# Phase 06 - Security

Per-phase security contract: threat register, accepted risks, and audit trail.

## Scope

Phase 06 migrated package management to Bun defaults while preserving npm/Node
fallback paths. This security review verifies the plan-time threat models from
`06-01-PLAN.md` and `06-02-PLAN.md` against the completed implementation,
summaries, and verification report.

The duplicate `T-06-SC` supply-chain entry appeared in both plans and is
deduplicated here as one phase-level supply-chain threat.

## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| Developer/CI shell to `package.json` scripts | Local and future CI callers execute script strings that resolve `bun`, `npm`, `node`, and root `concurrently` from PATH and installed dependencies. | Command execution and dependency installation intent |
| Public package registry to lockfiles/node_modules | Bun and npm resolve dependency tarballs from the registry into generated lockfiles and local installs. | Third-party package metadata and tarballs |
| Generated lockfiles to committed source | `bun.lock` and `package-lock.json` become trusted source artifacts used by future developers and CI. | Dependency graph and integrity metadata |
| Bun default path to npm/Node fallback path | Operational fallback depends on exact `:node` scripts and npm lockfiles remaining in sync with Bun defaults. | Runtime and install rollback command paths |
| Developer docs to shell execution | README and CLAUDE commands drive what humans and agents run locally. | Human-readable command contract |
| Bun build command to generated frontend bundle | `bun run build` invokes Vite and creates files served by the Express static host. | Generated frontend assets |
| Built frontend bundle to Bun unified server | `client/dist` crosses into `server/index.js` static serving via `FRONTEND_DIST=../client/dist`. | Public frontend HTML/CSS/JS |
| Verification report to future maintainers | The phase report becomes the source of truth for what was verified and which lockfiles/scripts are expected. | Audit evidence and maintenance instructions |

## Threat Register

| Threat ID | Category | Component | Disposition | Mitigation | Status |
|-----------|----------|-----------|-------------|------------|--------|
| T-06-01 | Tampering | `package.json` scripts | mitigate | Exact script contract exists in `package.json`: Bun defaults for install/dev/build/start and explicit npm/Node fallback aliases. Plan 06-01 verified script assertions. | closed |
| T-06-02 | Tampering | `bun.lock`, `package-lock.json`, `client/bun.lock`, `server/bun.lock` | mitigate | Root, client, and server Bun lockfiles are tracked; root/client/server npm fallback lockfiles are retained. Plan 06-01 verified Bun frozen installs and npm fallback installs. | closed |
| T-06-03 | Repudiation | Future dependency changes | mitigate | `package.json` includes `lockfiles:refresh`; README and CLAUDE document that dependency changes must regenerate and commit both lockfile families together. | closed |
| T-06-04 | Denial of Service | Local install/fallback commands | mitigate | Bun install path and npm fallback path were both validated for root, client, and server. `install:all:node`, `dev:node`, `build:node`, and `start:node` remain available. | closed |
| T-06-05 | Spoofing | Documentation command paths | mitigate | README and CLAUDE list Bun defaults first and exact npm/Node fallback commands second for install/dev/build/start. Phase 06 code review confirmed README and `package.json` match. | closed |
| T-06-06 | Tampering | `client/dist` generated by Bun-invoked Vite | mitigate | `bun run build` invokes `bun --cwd=client run build`, which runs the existing `vite build`; `BUN-PACKAGE-MANAGEMENT.md` records Vite retained. | closed |
| T-06-07 | Repudiation | Phase 6 verification evidence | mitigate | `BUN-PACKAGE-MANAGEMENT.md` records lockfile inventory, script contract, PKG-03 Vite build, static-serving smoke, and fallback build result. | closed |
| T-06-08 | Denial of Service | Fallback build path | mitigate | `npm run build:node` passed and is documented as the npm fallback build path. | closed |
| T-06-SC | Tampering | Package-manager/lockfile supply chain | mitigate | Only `concurrently@^9.2.0`, already present in the client dev dependency set, was promoted to root; both Bun and npm lockfiles freeze the dependency graph; frozen installs and npm fallback installs were verified. | closed |
