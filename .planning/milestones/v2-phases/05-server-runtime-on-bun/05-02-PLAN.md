<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Stripe -> Express webhook | Untrusted HTTP request carries signed raw body into billing fulfillment. |
| Smoke harness -> Express API | Test data crosses auth, middleware, and route boundaries through HTTP. |
| Runtime process -> MongoDB | Server runtime writes users, billing events, and audit/token data. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-02-01 | Spoofing | `/stripe/webhook` | mitigate | Use `constructEventAsync` with real `Stripe-Signature`; smoke includes valid and forged signed HTTP requests through Express. |
| T-05-02-02 | Tampering | Express raw body middleware order | mitigate | Preserve `/stripe/webhook` with `express.raw` before `express.json`; automated grep checks route ordering. |
| T-05-02-03 | Elevation of Privilege | `/api/generate-molecules` token middleware | mitigate | Smoke signs in through `/api/signin`, calls route with `Authorization: Bearer`, and asserts token decrement/exhaustion through MongoDB. |
| T-05-02-04 | Information Disclosure | Test secrets/env | mitigate | Use dummy `sk_test_dummy_key_never_calls_api`, test JWT secret, in-memory MongoDB, and no committed real keys. |
| T-05-02-05 | Denial of Service | Bun watch reload | mitigate | Watch smoke bounds startup/reload timeouts and kills child processes in cleanup. |
| T-05-02-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages are installed; smoke reuses existing `mongodb-memory-server`, `bcryptjs`, `mongodb`, and `stripe` dependencies. |
</threat_model>
